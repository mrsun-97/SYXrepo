一、算法设计与分析

题目： 用OpenMP实现PSRS排序

 

算法设计：

给定处理器数目，我们将长度为的数组分配给每个处理器，得到 ，之后每个处理器将数据分组，对位于的数据进行以下操作：

  • 快速排序：每个线程调用（见头文件）对自己负责的数据进行原地排序；
  • 正则采样：每个线程计算，并得到所有组内下标是 倍数的数据，然后根据自己写入正则采样数组 的相应位置。
  • 选择主元：使用语句#pragma omp master，使本步骤由主线程完成。主线程对数组中的数据排序，然后以为间隔，选出中间的个数据，放在数组的前个位置上。
  • 主元划分：每个线程都读取上述位置的个主元，以此将自己的数据分为段，准备接下来的全局交换

在全局交换时，我们需要交换的每个片段长短不一，为了方便交换，定义了以下结构体：

xxxxxxxxxx
typedef struct udfVec{        //slice of array
    data *pt;
    unsigned int len;
} Vec;

是一个 raw vector 指针加上长度信息。定义全局交换数组Vec trans[N][N]，在主元划分操作时，每个线程直接将划分好的第个片段信息写入中。

  • 全局交换：在上述主元划分时写操作结束后，每个线程直接读取中的信息，计算出自己获得的数据总长，申请内存空间，将这些信息对应的片段从原数组中拷贝到本地数组中。
  • 归并排序：每个线程调用merge_sort()（见头文件），利用原数组空间，将本地数组排序并按顺序放回原数组中。

四、备注：

在多次运行程序后，发现了一个小问题：3号线程分配到的数据一直是最多的。简单的分析可以发现，当我们正则采样时，我们取的样本是每个间隔的第一个数，是每个间隔中最小的，而且是向下取整，这样使得我们的主元相对于数据的-分位数偏小，因此数据有向后分配的趋势，最后都转移给了最后一个线程。

这个问题可以通过改变正则采样时选取每个间隔中的第几个数来解决，也可以通过改变选取主元位置的方式来解决。但不管怎么优化，由于数据的随机性，每个线程分配的数据总是会有多有少。
